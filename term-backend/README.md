## Описание

Python FastAPI Server TERM Backend — это комплексное серверное решение, разработанное на основе современного фреймворка FastAPI для управления проектами в области строительства и инженерии. Система предназначена для централизованного управления всеми аспектами строительных проектов, включая пользователей, документацию, оборудование и строительные объекты. Она интегрируется с AWS S3 для оптимального управления файлами и поддерживает широкий спектр функций, от мониторинга и логирования до управления состоянием сессий с использованием Redis.

Сервер использует PostgreSQL в качестве основной системы управления базами данных, обеспечивая надежное и масштабируемое хранение данных. Использование этой мощной реляционной базы данных позволяет обрабатывать сложные запросы, поддерживать транзакционную целостность и предоставлять высокий уровень безопасности данных. SQLAlchemy, интегрированный в проект как ORM, дополнительно упрощает разработку и обслуживание базы данных, делая взаимодействие с PostgreSQL более гибким и мощным.

TERM Backend является идеальным решением для компаний, стремящихся к цифровизации своих рабочих процессов, предоставляя инструменты для эффективного управления проектами, оптимизации рабочих процессов и повышения общей производительности команды. С помощью FastAPI, сервер обеспечивает высокопроизводительные, асинхронные API, которые могут масштабироваться для обработки запросов в реальном времени и интегрироваться с современными фронтенд-платформами для создания полноценных пользовательских интерфейсов.

## Основные функции

- **Управление пользователями**: Регистрация, обновление, удаление пользователей и управление их правами. Поддержка аутентификации и авторизации.
- **Управление группами**: Создание, обновление и удаление групп, а также управление членством в группах.
- **Интеграция с S3**: Загрузка, скачивание и управление файлами в AWS S3, включая поддержку больших файлов и защищённый доступ.
- **Мониторинг и логирование**: Регистрация событий системы для обеспечения аудита и возможности быстрого реагирования на инциденты.
- **Управление состоянием сессий**: Использование Redis для управления состоянием сессий и кэширования данных.
- **Безопасное хранение данных**: Все данные пользователей хранятся в зашифрованном виде в базе данных PostgreSQL.

## Технологии

- **FastAPI**: Асинхронный фреймворк для создания высокопроизводительных API с открытым исходным кодом.
- **SQLAlchemy**: Мощный ORM для управления базами данных через Python.
- **Alembic**: Инструмент для версионирования базы данных и проведения миграций.
- **Docker**: Используется для контейнеризации приложения, что облегчает развёртывание и масштабирование.
- **Redis**: Используется как брокер сообщений и для кэширования сессий и данных.
- **PostgreSQL**: Надёжная и мощная реляционная база данных для хранения всех данных приложения.
- **MinIO**: Объектное хранилище, совместимое с Amazon S3, используется для хранения файлов и артефактов.

## Локальная инфраструктура с Docker Compose

Проект использует `docker-compose` для локальной разработки и тестирования. В `docker-compose` файле настроены следующие сервисы:

- **term-backend**: Основное приложение FastAPI.
- **redis-local**: Используется для кэширования и управления сессиями.
- **postgres-local**: База данных PostgreSQL для хранения данных.
- **minio-local**: Локальное хранилище, совместимое с S3, для работы с файлами.

## Зависимости

Проект использует `Poetry` для управления зависимостями. Полный список зависимостей находится в файле `pyproject.toml`.

## Запуск

Для успешного запуска и работы проекта следуйте приведенным ниже шагам:

### Шаг 1. Проверка предварительной установки необходимых инструментов

Убедитесь, что на вашем компьютере установлены `make`, `Docker` и `Docker Compose`. Инструкции по установке смотрите разделе в [Установка Make, Docker и Docker Compose](#установка-make-docker-и-docker-compose)

### Шаг 2. Запуск сервисов с помощью Docker

Запустите все необходимые сервисы, используя Docker Compose:

```bash
make docker.service.compose.run
```

Эта команда запустит все сервисы, определенные в файле `docker/docker-compose.local.yml`. Дождитесь полной инициализации всех контейнеров.

### Шаг 3. Выполнение миграций в Docker

Примените миграции к базе данных, выполнив следующую команду:

```bash
make alembic.remote.upgrade
```

Это обновит структуру базы данных до последней версии, предусмотренной миграциями.

### Шаг 4. Первичная настройка системы

После выполнения миграций, система автоматически создаст учетную запись суперпользователя. Обратите внимание на консоль, где будет отображена информация для входа:

```bash
Admin account: email['admin@example.com'] password['initialPassword123']
```

Эти учетные данные используются для первого входа в систему. Рекомендуем изменить пароль сразу после первого входа для обеспечения безопасности.

### Шаг 5. Проверка работоспособности системы

После запуска сервисов и выполнения миграций проверьте доступность вашего API по адресу `http://localhost:5000/`. Используйте Swagger UI (`http://localhost:5000/docs`) или ReDoc (`http://localhost:5000/redoc`) для взаимодействия с API и тестирования различных эндпоинтов.

## Взаимодействие с API

После запуска сервера, API будет доступен по адресу `http://localhost:5000/`. Вы можете взаимодействовать с API и изучать его возможности через следующие встроенные инструменты документации:

### Swagger UI

Swagger UI предоставляет интерактивную веб-страницу, где вы можете просматривать все доступные эндпоинты, их документацию и отправлять запросы к API без написания дополнительного кода.

- **Доступ**: Откройте `http://localhost:5000/docs` в вашем браузере.

### ReDoc

ReDoc предоставляет ещё один вариант документации, который может быть предпочтительнее за счёт своего чистого и простого интерфейса.

- **Доступ**: Перейдите на `http://localhost:5000/redoc` в вашем веб-браузере.
