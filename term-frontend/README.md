## Описание

Nuxt 3 SSR Term Frontend — это веб-приложение на базе фреймворка Vue 3 и Nuxt 3, созданное для того, чтобы пользователи могли быстро просматривать контент. Приложение осуществляет рендеринг веб-страниц на стороне сервера, что обеспечивает их быстрое отображение в браузере пользователя.

## Основные функции

- **Управление пользователями**: Регистрация, обновление, удаление пользователей и управление их правами. Поддержка аутентификации и авторизации.
- **Управление группами**: Создание, обновление и удаление групп, а также управление членством в группах.
- **Добавление и просмотр BIM-моделей**: Загрузка и просмотр BIM-моделей в формате `*.ifc`, а также их кэширование на сервере для более быстрого просмотра.
- **Мониторинг и логирование**: Регистрация событий системы для обеспечения аудита и возможности быстрого реагирования на инциденты.

## Технологии

- **Vue 3**: Фреймворк для создания пользовательских интерфейсов.
- **Nuxt 3**: Фреймворк с открытым исходным кодом, который позволяет создавать высокопроизводительные веб-приложения и веб-сайты с полным стеком производственного уровня.
- **Nuxt UI**: Библиотека пользовательского интерфейса для современных веб-приложений.
- **Pinia**: Библиотека для храения глобальных состояний веб-приложения.
- **OpenBIM Components**: Библиотека. которая представляет собой набор инструментов BIM, основанных на Three.js. Он включает готовые функции для простого создания браузерных 3D BIM-приложений.
- **Three.js**: Библиотека, используемая для создания и отображения анимированной компьютерной 3D графики при разработке веб-приложений.

## Локальная инфраструктура с Docker Compose

Проект использует `docker-compose` для локальной разработки и тестирования. В `docker-compose` файле настроены следующие сервисы:

- **term_frontend**: Основное приложение Nuxt.
- **term_nginx**: HTTP-сервер и обратный прокси-сервер.

## Зависимости

Проект использует `Node Package Manager (npm)` для управления зависимостями. Полный список зависимостей находится в файле `package.json`.

## Запуск

Для успешного запуска и работы проекта следуйте приведенным ниже шагам:

### Шаг 1. Проверка предварительной установки необходимых инструментов

Убедитесь, что на вашем компьютере установлены `make`, `Docker` и `Docker Compose`. Инструкции по установке смотрите разделе в [Установка Docker и Docker Compose](#установка-docker-и-docker-compose)

### Шаг 2. Запуск сервисов с помощью Docker

Запустите все необходимые сервисы, используя Docker Compose:

```bash
docker compose -f "docker-compose.yml" up -d --build
```

Эта команда запустит все сервисы, определенные в файле `docker-compose.yml`. Дождитесь полной инициализации всех контейнеров.

### Шаг 3. Проверка работоспособности системы

После запуска сервисов проверьте доступность вашего Веб-приложения по адресу `http://localhost/`.
